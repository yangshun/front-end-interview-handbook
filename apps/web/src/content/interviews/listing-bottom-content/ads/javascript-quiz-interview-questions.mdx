### Example JavaScript Quiz Questions

Here are some examples of the types of questions you might face in a JavaScript interview, categorized by topic.

#### 1. Accessibility

- **Question:** What are ARIA roles, and how do they help in web development?
  - _Answer:_ ARIA (Accessible Rich Internet Applications) roles provide additional information about elements on a webpage, helping screen readers and assistive technologies better interpret the content. For example, using `role="button"` on a div element ensures assistive technologies treat it as a button.
- **Question:** How can you ensure that your JavaScript application is accessible to users with disabilities?
  - _Answer:_ Use semantic HTML elements (e.g., `&lt;button>` for buttons, `&lt;header>` for headers), implement ARIA attributes where necessary, test with screen readers, and ensure proper keyboard navigation.

#### 2. Internationalization (i18n)

- **Question:** What is the purpose of the `Intl` object in JavaScript?
  - _Answer:_ The `Intl` object provides language-sensitive string comparison, number formatting, and date/time formatting. For example, `Intl.NumberFormat` formats numbers based on a specific locale.
- **Question:** How can you dynamically change the language of your application using JavaScript?
  - _Answer:_ Store the user's language preference in a variable or localStorage and use it to fetch translations. Update the text content dynamically when the language setting changes.

#### 3. Performance

- **Question:** What are some ways to improve the performance of a JavaScript application?
  - _Answer:_ Minify and bundle JavaScript files, use lazy loading for resources, optimize DOM manipulation, cache data efficiently, and debounce or throttle event listeners.
- **Question:** How does `requestAnimationFrame` improve performance when creating animations?
  - _Answer:_ `requestAnimationFrame` synchronizes the execution of animations with the refresh rate of the browser, making animations smoother and more efficient by reducing unnecessary reflows and repaints.

#### 4. Networking

- **Question:** What is the difference between `fetch()` and `XMLHttpRequest` in JavaScript?
  - _Answer:_ `fetch()` is a modern, Promise-based API for making network requests, while `XMLHttpRequest` is an older, callback-based API. `fetch()` is more concise, supports streaming responses, and has better error-handling capabilities.
- **Question:** How can you handle errors effectively when using the `fetch()` API?

_Answer:_ Use `.catch()` to handle network errors and check the `response.ok` property to detect HTTP errors. For example:

```js
fetch('/api/data')
  .then((response) => {
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }

    return response.json();
  })
  .catch((error) => console.error('Fetch error:', error));
```

#### 5. Security

- **Question:** How can you prevent Cross-Site Scripting (XSS) attacks in your JavaScript code?
  - _Answer:_ Sanitize user inputs, escape output data, use Content Security Policies (CSP), and avoid directly inserting user-generated content into the DOM.
- **Question:** What is the purpose of `strict mode` in JavaScript, and how does it improve security?
  - _Answer:_ `strict mode` enforces stricter parsing and error handling, helping to catch common coding errors and prevent unsafe actions, such as using undeclared variables.

#### 6. Testing

- **Question:** What is the difference between unit testing and integration testing in JavaScript?
  - _Answer:_ Unit testing focuses on testing individual functions or components in isolation, while integration testing evaluates how different parts of the application work together.
- **Question:** How can you test asynchronous JavaScript code?

_Answer:_ Use testing libraries like Jest or Mocha that support Promises and async/await. For example:

```js
test('fetches data correctly', async () => {
  const data = await fetchData();

  expect(data).toBeDefined();
});
```
