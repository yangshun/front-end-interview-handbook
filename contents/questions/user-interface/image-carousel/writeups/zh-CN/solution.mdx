## 解决方案

该解决方案可以分解为两个部分：

1. **渲染**：按钮的布局和定位
2. **状态管理**：状态设计和状态变化

### 渲染

图片轮播布局由 3 部分组成：

#### 1. 图片轮播容器

图片轮播的根元素将负责整体尺寸，并包含显示的图像和按钮元素。由于高度是固定的，我们可以在根元素上设置 `height: 400px`。但是，宽度是流动的，在小于 600px 的屏幕上，它应该缩小以适应屏幕。对于这种行为，我们可以使用 CSS [`min`](https://developer.mozilla.org/en-US/docs/Web/CSS/min) 函数（例如 `min(600px, 100vw)`）。

#### 2. 显示的图片

可以使用普通的 `<img>` 标签和 `object-fit: contain` 渲染图像，这将调整图像大小以适应 600px x 400px 的可用空间，同时保持图像的固有纵横比。

#### 3. 导航按钮

各种按钮都位于图片轮播中的固定位置。这是通过在图片轮播的根元素上添加 `position: relative`，并在按钮元素上添加 `position: absolute` 来实现的。

要垂直居中 prev/next 按钮，请使用 `top: 50%; transform: translateY(-50%)` 的组合。`top: 50%` 将元素定位为具有容器元素 50% 的垂直偏移，而 `translateY(-50%)` 将元素向上移动元素高度的一半。页面按钮水平居中，可以采用类似的方式。

### 状态

图片轮播所需的最小状态是当前图片索引。只需要一个状态值。

#### 状态变化

* **Next 按钮**：将 `currIndex` 增加 1。
* **Prev 按钮**：将 `currIndex` 减少 1。
* **页面按钮**：直接将 `currIndex` 设置为页码。

为了启用循环行为，我们需要确保 `currIndex` 始终介于 `0` 和 `images.length - 1` 之间。我们可以通过将 `images.length` 添加到任何值，并使用模运算符 (`%`) 来实现这一点。
