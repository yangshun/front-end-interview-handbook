**注意：**这是[Accordion](/questions/user-interface/accordion)的高级版本，在尝试此问题之前，您应该先完成该问题。

> 手风琴是一组垂直堆叠的交互式标题，每个标题都包含一个标题、内容片段或代表内容部分的缩略图。标题充当控件，使用户能够显示或隐藏其相关的内容部分。手风琴通常用于减少在单个页面上呈现多个内容部分时对滚动的需求。

*来源：[手风琴模式](https://www.w3.org/WAI/ARIA/apg/patterns/accordion/)*

在[Accordion](/questions/user-interface/accordion)中，我们构建了一个可以展开/折叠每个部分内容的实用手风琴组件。但是，构建好的 UI 组件不仅仅是功能，我们还必须通过向 DOM 元素添加正确的 ARIA 角色、状态和属性来确保我们的组件也具有出色的可访问性。

## 要求

ARIA 创作实践指南有一个[关于要添加到手风琴的各个元素的 ARIA 角色、状态和属性的长期指导方针列表](https://www.w3.org/WAI/ARIA/apg/patterns/accordion/)。我们应该为这个问题实现以下（即兴）指南：

* 每个手风琴标题的标题都包含在 `<button>` 元素中。
* 如果与手风琴标题关联的手风琴面板可见，则标题按钮元素将 `aria-expanded` 设置为 `true`。如果面板不可见，则 `aria-expanded` 设置为 `false`。
* 手风琴标题按钮元素将 `aria-controls` 设置为包含手风琴面板内容的元素的 ID。
* 每个用作面板内容容器的元素都具有角色 `region` 和 `aria-labelledby`，其值引用控制面板显示的按钮。

## 笔记

框架代码使用[Accordion](/questions/user-interface/accordion)的解决方案，但您可以自由使用自己的解决方案作为起点。
