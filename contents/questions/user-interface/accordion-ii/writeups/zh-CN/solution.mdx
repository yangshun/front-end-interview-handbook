## 解决方案

除了添加正确的 ARIA 角色和状态（这很简单）之外，我们还需要将手风琴标题与相应的手风琴内容/面板链接起来。因此，我们创建了两个函数 `getAccordionHeaderId` 和 `getAccordionPanelId` 来执行此操作。

* `getAccordionHeaderId` 生成一个唯一的 ID 字符串，用作标题元素的 `id` 属性的值。此 ID 将用作相应手风琴面板的 `aria-labelledby` 属性的值。
* `getAccordionPanelId` 生成一个唯一的 ID 字符串，用作手风琴面板的 `id` 属性的值。此 ID 将用作相应手风琴标题的 `aria-controls` 属性的值。

由于页面上可能存在多个 `Accordion` 组件实例，并且我们无法保证手风琴部分的值是全局唯一的，因此每个 `Accordion` 实例都需要一个唯一的标识符。`useId` React 钩子可用于为每个 `Accordion` 实例生成唯一的 ID。最终的 ID 字符串将是 `Accordion` 实例的 ID、项目值以及它是标题还是面板的串联。
