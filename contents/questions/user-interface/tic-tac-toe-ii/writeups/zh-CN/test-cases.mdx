## 测试用例

1. 初始棋盘状态
   * 验证游戏棋盘是否正确显示，包含 N x N 个空单元格。
   * 验证状态消息是否指示轮到玩家 X。
2. 标记单元格
   * 点击一个空单元格。
   * 验证该单元格是否标记为“X”，并且轮到玩家 O。
   * 点击另一个空单元格。
   * 验证第二个单元格是否标记为“O”，并且轮到玩家 X。
   * 重复此过程，再点击几个单元格，以确保标记正确放置，并且轮流在玩家之间切换。
3. 获胜条件
   * 通过在行、列或对角线上标记 M 个单元格，为玩家 X 或玩家 O 创建一条获胜线。
   * 验证状态消息是否正确显示获胜玩家的消息。
   * 验证在游戏获胜后，不能再进行任何标记。
   * 点击“重置”按钮。
   * 验证棋盘是否已清除，并且游戏重新开始，轮到玩家 X。
4. 平局条件
   * 在棋盘上填充所有单元格，而没有创建获胜线。
   * 验证状态消息是否正确显示平局消息。
   * 验证平局后，不能再进行任何标记。
   * 点击“重置”按钮。
   * 验证棋盘是否已清除，并且游戏重新开始，轮到玩家 X。
5. 辅助功能
   * 使用屏幕阅读器导航并与游戏交互。
   * 验证状态消息、单元格标记和按钮是否被屏幕阅读器正确播报。
   * 验证游戏是否可以使用键盘导航进行游戏和重置。
6. 响应式设计
   * 在不同的屏幕尺寸（例如桌面、平板电脑和移动设备）上测试游戏。
   * 验证游戏棋盘和单元格是否正确调整到不同的屏幕尺寸。
   * 确保游戏在较小的屏幕上仍然可以玩并且具有视觉吸引力。
