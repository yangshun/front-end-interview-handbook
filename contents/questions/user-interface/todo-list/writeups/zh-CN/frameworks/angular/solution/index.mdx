## 解决方案

要添加新任务，我们需要在输入字段中输入文本，然后单击“提交”按钮。随后，将调用 `addTask()` 方法。如果提供的字符串不为空，它会将该元素添加到 `tasks`。添加任务后，我们清除 `newTask` 变量。

在模板中显示任务列表时，我们还为它们分配了一个按钮，通过调用 `deleteTask(task.id)` 来删除任务，该方法仅使用指定的 ID 过滤掉给定的任务。

### 状态

* `id` - 仅用于示例 id 目的的变量。它用于递增任务中的 id，以避免重复
* `newTask` - 通过使用 ngModel `[(ngModel)]="newTask"` 与输入绑定的 newTask 的名称。
* `tasks` - 我们所有的任务对象

### 模型

```typescript
export interface Task {
  id: number;
  label: string;
}
```

### Angular 见解

* 您可以使用更高级的、类似反应式的方法，使用 RxJS（无需使用订阅）。但是，如果您选择订阅，请记住销毁它们以防止内存泄漏。

* 您可以创建用于维护状态的服务。在智能组件中注入服务，并创建仅显示和与 UI 交互的哑组件，并且哑组件的所有事件都传递给智能组件。

* 如果您对最新的 Angular 版本有信心，请考虑使用信号独立 API。

* 您可以尝试更多地关注诸如 `readonly`、`private`、`public` 和 `void` 等关键字。

* 如果您正在创建更大的应用程序，最好为每个组件使用样式，而不是将所有样式放在一个文件中。
