## 解决方案

### 响应式状态

我们需要两个响应式状态：`tasks` 和 `newTask`。

`tasks`：由于存在可以修改的任务列表，我们需要将其作为组件的响应式状态的一部分。在 Vue 中渲染元素列表时，我们可以使用 `v-for`，并且[建议](https://vuejs.org/guide/essentials/list.html#maintaining-state-with-key)尽可能为每个项目提供一个 `key`。我们不能使用任务的文本作为 `key`，因为它们不能保证是唯一的。对于这个问题，我们使用自动递增计数器为每个任务生成一个唯一的索引。最可靠的方法是为每个任务生成一个唯一的 ID。像 [`uuid`](https://github.com/uuidjs/uuid) 这样的库就会派上用场。

`newTask`：一个绑定到新任务输入字段的响应式状态。要了解更多信息，您可以参考[表单输入绑定](https://vuejs.org/guide/essentials/forms.html#text)的官方文档。

### 添加任务

新任务应添加到 `tasks` 数组的末尾。我们可以使用新的 `id` 和 `label` 字段构造一个新的任务对象，并推送到 `tasks` 响应式状态。

### 删除任务

为每个任务对象提供唯一的 `id` 可以简化这里的事情，因为我们可以过滤现有列表并排除与要删除的 `id` 对应的任务。
