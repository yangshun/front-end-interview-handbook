## 可访问性

模态对话框有许多可访问性考虑因素，在面试期间可能无法实现太多。

为了让您了解，以下是一些更重要的考虑因素：

* ARIA 属性：
  * 在模态容器上使用 `role="dialog"`。
  * 添加 `aria-modal="true"` 以表明模态框会阻止与页面内容的交互。
  * 使用 `aria-labelledby` 将模态框的标题与对话框关联起来。
  * 如果有额外的描述性文本，请使用 `aria-describedby`。
* 键盘导航：
  * 确保可以使用键盘控件打开和关闭模态框。
  * 提供一种使用 <kbd>Esc</kbd> 键关闭模态框的方法。
* 焦点管理：
  * 在模态框打开时将焦点设置为模态框中第一个可聚焦的元素。
  * 在模态框打开时将焦点限制在模态框内，防止用户从模态框外进行 Tab 操作。
  * 在模态框关闭时将焦点返回到触发元素。
