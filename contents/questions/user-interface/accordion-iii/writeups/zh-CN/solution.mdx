## 解决方案

### 监听键盘事件

首先，了解 `keypress` 和 `keydown` 事件之间的区别至关重要。`keypress` 事件仅在按下产生字符值的键时触发。这将排除 <kbd>Up</kbd>、<kbd>Down</kbd>、<kbd>Home</kbd> 和 <kbd>End</kbd> 键，而这些是此问题的要求。`keypress` 事件也已弃用，不应使用。由于这些原因，我们应该使用 `keydown` 事件。我们将向根 `<div>` 元素添加一个 `keydown` 事件侦听器。

要知道按下了哪个键，我们可以在传递给 `keydown` 事件回调的事件上使用 `event.key` 或 `event.code`。[`event.key` 与 `event.code` 之间存在一些差异](https://javascript.info/keyboard-events)，但就此问题而言，这无关紧要，可以忽略。我们只使用 `event.code`。

### 响应键盘事件

请注意，只有当焦点当前位于其中一个手风琴标题上时，才应响应这些键盘事件。我们可以使用 `document.activeElement` 获取页面上当前聚焦的 DOM 元素，并检查它是否具有 `data-accordion-value` 属性，该属性已添加到所有手风琴标题按钮。

接下来，我们读取 `event.code` 属性，并根据其值使用自定义代码进行响应。`switch` 语句适用于这种情况：

* `ArrowUp`：聚焦到上一个标题，如果焦点位于第一个标题上，则“环绕”到最后一个标题。我们首先找到当前聚焦标题的索引，将其减一，并使用模运算来优雅地处理“环绕”。
* `ArrowDown`：聚焦到下一个标题，如果焦点位于最后一个标题上，则“环绕”到第一个标题。我们首先找到当前聚焦标题的索引，将其加一，并使用模运算来优雅地处理“环绕”。
* `Home`：聚焦到第一个标题。
* `End`：聚焦到最后一个标题。

一个有效的键盘事件应该聚焦于一个新的标题按钮。由于我们知道每个标题的 `id`，我们可以通过使用 `document.getElementById()` 和所需的标题元素 id 对其进行命令式调用 `.focus()`。这种方法不符合惯例，但在面试期间是可以接受的。
