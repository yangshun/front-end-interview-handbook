## 解决方案

最广泛支持的解决方案是基于 Flexbox 的解决方案。它具有出色的浏览器支持，并且最容易理解。基于 Grid 的方法也越来越受欢迎，但不如 Flexbox 那样受到支持。

使用 flexbox 实现规范主要有几个部分。让我们深入研究每一个。

### 粘性页脚

Holy Grail 布局问题还包含另一个经典问题：当没有足够的内容来填满页面时，使页脚粘在屏幕底部。

这可以通过将 `min-height: 100vh` 添加到页面内容的容器来解决。由于直接子元素将以垂直方式布局，因此我们还向该元素添加 `display: flex` 和 `flex-direction: column`。

页眉和页脚是固定高度，列是可变高度，旨在填满所有剩余空间。为了实现这一点，将 `flex-grow: 1` 添加到包装列的 `<div>` 中。

### 列

使所有列等高的要求也可以通过 Flexbox 轻松解决。通过将 `display: flex` 添加到列的 `div` 包装器，即可满足此要求。

像以前一样，可以使用 `flex-grow: 1` 实现灵活宽度的 `<main>` 内容部分，它将填满所有可用的水平空间。

必须将 `flex-shrink: 0` 添加到 `<nav>` 和 `<aside>`，这样当 `<main>` 中的内容太宽时，它们就不会缩小。
