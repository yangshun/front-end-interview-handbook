**注意：**这是[模态对话框](/questions/user-interface/modal-dialog)的高级版本，在尝试此问题之前，您应该先完成该问题。

> 对话框是覆盖在主窗口或其他对话框窗口上的一个窗口。模态对话框下的窗口是不活动的。也就是说，用户无法与活动对话框窗口之外的内容进行交互。活动对话框之外的非活动内容通常在视觉上被遮蔽或变暗，因此难以辨别，并且在某些实现中，尝试与非活动内容交互会导致对话框关闭。

*来源：[对话框（模态）模式 | ARIA 创作实践指南](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/)*

在[模态对话框](/questions/user-interface/modal-dialog)中，我们构建了一个功能性模态对话框组件，该组件可以通过自定义标题和内容进行打开和关闭。

然而，构建良好的 UI 组件不仅仅是功能，我们还必须通过向 DOM 元素添加必要的 ARIA 角色、状态和属性，以及键盘交互和焦点管理来确保我们的组件具有出色的可访问性。

在本问题中，我们将重点关注实现必要的 ARIA 角色、状态和属性。

## 要求

ARIA 创作实践指南有一个[关于要添加到模态对话框组件的各种元素的 ARIA 角色、状态和属性的长的指导方针列表](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/)

为本问题实现以下准则：

* 充当对话框容器的元素具有 `dialog` 角色。
* 操作对话框所需的所有元素都是具有 `dialog` 角色的元素的后代。
* 对话框容器元素将 `aria-modal` 设置为 `true`。
* 对话框为 `aria-labelledby` 属性设置了值，该属性引用可见的对话框标题。
* 在具有对话框角色的元素上设置 `aria-describedby` 属性，以指示对话框中哪些元素包含描述对话框的主要目的或消息的内容。
  * **注意**：如果对话框内容包含语义结构（例如列表、表格或多个段落），则建议省略指定 `aria-describedby`，这些结构需要被感知才能轻松理解内容，也就是说，如果内容在被声明为单个不间断的字符串时难以理解。但是，为了练习，我们仍将实现此属性。

## 注意事项

框架代码使用[模态对话框](/questions/user-interface/modal-dialog)的解决方案，但您可以自由使用自己的解决方案作为起点。

[`<dialog>` HTML 元素](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dialog)可用，但为了练习，请在不使用该元素的情况下实现模态对话框。
