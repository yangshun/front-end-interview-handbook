## 解决方案

该解决方案可以分解为几个部分：

1. 收集表单数据。
2. 执行操作。
3. 显示最终结果。

### 状态

我们需要几条信息来跟踪应用程序的状态：

* `selected`：所选用户的 id 的状态变量
* `search`：搜索词的状态变量
* `first`：名字的状态变量
* `last`：姓氏的状态变量
* `users`：用户列表的状态变量

### 收集表单数据

我们使用几个输入字段来收集搜索词、名字和姓氏。搜索词用于过滤用户列表。名字和姓氏用于创建/更新用户。

通过在名字和姓氏上使用 `<input required />`，我们确保了这些字段已通过验证，并且不能为空。

### 执行操作

当用户被选中时，名字和姓氏字段会使用所选用户的数据进行更新。如果没有选定的用户，这些字段为空。

我们使用简单的按钮来创建/更新/删除/取消操作：

* 创建：如果没有选定的用户，并且名字和姓氏不为空，我们会创建一个新用户并将其添加到用户列表中。否则，该按钮将被禁用。
* 更新：如果有一个选定的用户，并且新的名字和姓氏不为空，我们会更新所选用户的名字和姓氏。否则，该按钮将被禁用。
* 删除：如果有一个选定的用户，我们会从用户列表中删除所选用户。否则，该按钮将被禁用。
* 取消：我们取消选择当前选定的用户。如果没有选定的用户，该按钮将被禁用

### 显示最终结果

我们使用一个简单的选择字段来显示用户列表。选择字段的值是 `selected` 状态变量。用户列表是根据搜索词计算的。
