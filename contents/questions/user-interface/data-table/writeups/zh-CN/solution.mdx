## 解决方案

在前端面试中经常会问到数据表，因为在几乎每家公司都会构建一个带有过滤功能的 UI，用于显示分页数据。

由于已经提供了表格框架，我们可以专注于数据表的状态和数据操作方面。

### 状态

状态很简单。只需要两个状态值，即当前页和页面大小。由于在这种情况下数据不会改变，因此用户数据不需要成为状态的一部分。

这些状态值由页面大小 `<select>` 和上一个/下一个按钮操作。

最大页数可以从用户数除以页面大小得出，因此它不需要成为状态的一部分。

### 分页

我们实现了一个函数 `paginateUsers`，它接受用户列表、页码和页面大小。它将返回当前页的用户列表和总页数。

要确定当前页的用户列表，我们可以确定起始和结束索引，然后使用 `Array.prototype.slice()` 从用户列表中提取适当的切片：

* **起始索引**：将页码乘以页面大小。由于页码从 1 开始，我们首先必须从页码中减去 1。此值将是要切片的用户的起始索引。
* **结束索引**：将页面大小添加到计算出的起始索引。由于我们将使用 `Array.prototype.slice()`，因此结束索引是否超过列表的大小无关紧要。

`paginateUsers()` 将在渲染路径中被调用，并且返回的用户 `pageUsers` 是要渲染的当前用户页面。渲染代码不需要更改太多。

### 用户体验

我们可以进行的其他一些用户体验改进：

* 当页面大小更改时，将页码重置为第一页。否则，当用户更改页面大小并且当前页面超过总页数且未显示任何内容时，可能会造成混淆。
* 当无法导航到上一页或下一页时，禁用上一个/下一个按钮。
