## 测试用例

让游戏配置为 `PLAYERS` 玩家，获胜条件为 `COUNT_TO_WIN` 个相同颜色的连续圆盘。

对于给定的 `ROWS`-by-`COLUMNS` 网格 `grid` 和给定的 `player`，

* 对于任何给定的列 `i`：
  * 如果至少有一个空单元格 `cell`，则放置一个圆盘将导致 `cell` 被填充为 `player` 的圆盘颜色。
  * 在一列中放置一个圆盘将从底部填充空间。
  * 如果没有空单元格，则无法在该列中放置圆盘。 如果您对此情况有其他行为，也希望它发生。
* 当一个圆盘被放入某个单元格 `cell` 时，对于穿过 `cell` 的四个方向：─、│、╲ 和 ╱，
  * 如果有至少 `COUNT_TO_WIN` 个连续的单元格填充了 `player` 的圆盘颜色，则会有一个获胜玩家的指示器。
  * 否则，在下一轮中放置的圆盘的颜色是 `PLAYERS` 中的下一个玩家的颜色。
* 游戏进度：
  * 玩家可以轮流放置圆盘，直到一名玩家获胜或出现平局。
  * 如果所有单元格都已填满且没有玩家获胜，则会出现平局指示器。
* 游戏结束
  * 当出现获胜或平局指示器时，将无法再放置圆盘。
* 单击“重置”按钮会将所有单元格 `grid` 变为其基本颜色，并清除任何获胜或平局指示器。
