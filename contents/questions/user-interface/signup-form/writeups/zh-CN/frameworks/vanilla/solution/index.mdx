由于浏览器具有原生表单验证功能，我们将使用它，这样我们就可以编写更少的 JavaScript。

基于 HTML 的验证不允许自定义错误样式。因此，如今，许多表单选择基于 JavaScript 的验证以更好地控制验证样式也就不足为奇了。在面试环境中，时间限制很紧。如果对特定验证样式没有要求，使用基于 HTML 的验证将为您节省一些时间，并向面试官展示您的 HTML 知识，这是一个积极的信号。

如果您还不熟悉基于 HTML 的验证，这个问题是一个很好的机会来熟悉它。如果使用基于 HTML 的验证，我们只需要编写很少的 JavaScript。只有匹配的密码确认必须使用 JavaScript 完成。

`<input>` 上的以下属性对于在提交发生之前验证表单值特别有用：

* `required`：输入值不能为空。
* `pattern`：输入值必须与指定的正则表达式匹配。
* `minlength`：输入值必须包含最少的字符数。

有[更多验证属性](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Constraint_validation)，但以上是与注册表单相关的属性。

所有字段都是 `required`，因此我们可以将此属性添加到所有 `<input>` 中。

* 用户名：要验证至少 4 个字符，请使用 `minlength="4"`。要验证字母数字约束，我们使用简单的正则表达式 `^[a-zA-Z0-9]+$`。
* 电子邮件：通过使用 `type="email"`，如果该值不为空，浏览器将验证该值是否与电子邮件格式匹配。
* 密码/确认密码：使用 `minlength="6"` 进行最小长度验证，并使用 `type="password"` 隐藏值。

由于我们还需要验证密码是否匹配，我们可以向表单添加一个 `'submit'` 事件侦听器，使用 `event.preventDefault()` 拦截浏览器提交，并使用 JavaScript 进行验证。如果密码验证失败，我们会在密码确认字段下方显示错误消息。
