**注意：** 这是 [图片轮播](/questions/user-interface/image-carousel) 问题的后续问题，在尝试此问题之前，您应该先完成该问题。

构建一个图片轮播，显示一系列图片，在图片之间切换时平滑过渡。

此问题与 [图片轮播 II](/questions/user-interface/image-carousel-ii) 的主要区别在于，在任何时候 DOM 中应该**最多只有两个图像元素**。这提高了性能，因为 DOM 中存在的图像元素数量不再与图像数量成正比，并且图片轮播不会不必要地加载未显示的图像。但是，这种方法会增加实现的复杂性。

## 要求

* 应支持 [图片轮播](/questions/user-interface/image-carousel) 问题的全部要求。
* 单击“上一个/下一个”按钮时，当前图像应平滑地移出，而下一张图像进入。转换方向分别为向左/向右。
* 单击页面按钮时的过渡是可选的，但强烈建议。 旨在重用“上一个/下一个”按钮的过渡代码。
* 在任何时候 DOM 中应该**最多只有两个图像元素**。

单击“查看您正在构建的内容”按钮以查看结果的预览。

## 笔记

骨架代码使用 [图片轮播](/questions/user-interface/image-carousel) 的解决方案，但您可以自由使用自己的解决方案作为起点。
