React Hooks were introduced in React 16.8 and they allow developers to use state and lifecycle features in function components. Hooks simplify code organization, reduce boilerplate, and enable better logic reuse.

## Common hooks

### `useState`

`useState` enables function components to manage state without needing a class component.

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>Count: {count}</button>;
}
```

- **When to use**: When a component needs local state that is contained within the component instance and not meant to be shared with siblings/parents
- **Pitfalls**:
  - Directly modifying state instead of using the setter function (`setState`)
  - Referencing stale values in closures and updating with stale values

### `useEffect`

`useEffect` handles side effects, such as fetching data, subscribing to events, or interacting with the DOM.

```jsx
import { useState, useEffect } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('/api/data')
      .then((response) => response.json())
      .then(setData);
  }, []);

  return <div>{data ? JSON.stringify(data) : 'Loading...'}</div>;
}
```

- **When to use**: When handling side effects like API calls, subscribing to external events (e.g. DOM resize)
- **Pitfalls**:
  - Forgetting to add dependencies in the dependency array
  - Causing infinite loops by updating state inside `useEffect` without proper dependency management

`useEffect` is primarily for side effects / interacting with external services which isn't too common during interviews. If you find yourself reaching for `useEffect` in your solutions, consider again if there are alternatives.

### `useContext`

`useContext` provides a way to share state across components without prop drilling.

```jsx
import { createContext, useContext } from 'react';

const ThemeContext = createContext('light');

function ThemedComponent() {
  const theme = useContext(ThemeContext);
  return (
    <div style={{ background: theme === 'dark' ? '#333' : '#fff' }}>
      Theme: {theme}
    </div>
  );
}
```

- **When to use**: For global state like themes, authentication, or user settings
- **Pitfalls**:
  - Overusing `useContext` for deeply nested state that might be better managed with a state management library
  - Using it without memoization, leading to unnecessary re-renders

### `useRef`

`useRef` stores a mutable reference that persists across renders, commonly used for accessing DOM elements or persisting state without causing re-renders.

```jsx
import { useRef } from 'react';

function FocusInput() {
  const inputRef = useRef(null);

  return (
    <div>
      <input ref={inputRef} />
      <button onClick={() => inputRef.current.focus()}>Focus Input</button>
    </div>
  );
}
```

- **When to use**: When interacting with the DOM or for storing data that does not affect the UI of the component
- **Pitfalls**: Overusing `useRef` for state instead of `useState`

## Rules of hooks

React has strict rules to ensure hooks work correctly:

1. **Only call hooks at the top level**: Do not call hooks inside loops, conditions, or nested functions
2. **Only call hooks from React function components or custom hooks**: Avoid using hooks in regular JavaScript functions
3. **Follow the rules for dependencies**: It's a good practice to provide a dependency array in `useEffect`, `useCallback`, and `useMemo` to avoid unexpected behavior. However in future the [React compiler](https://react.dev/learn/react-compiler) aims to remove the need for writing `useCallback` and `useMemo` manually

## Custom hooks

Custom hooks allow logic reuse across components while keeping them clean and modular.

### When and why to abstract logic

- **Encapsulation**: Extract complex logic from components to keep them concise.
- **Reusability**: Share logic across multiple components without duplication.
- **Readability**: Improve code maintainability by separating concerns.

Example:

```jsx
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then((res) => res.json())
      .then((data) => {
        setData(data);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
}
```

### Best practices for building and testing custom hooks

- **Prefix with `use`**: Ensures React recognizes it as a hook.
- **Keep dependencies minimal**: Avoid unnecessary dependencies in `useEffect`.
- **Test behavior, not implementation**: Use libraries like React Testing Library to test custom hooks in isolation.

## Hooks anti-patterns

Avoid these common mistakes when using hooks:

### 1. Updating state without considering previous state

```jsx
setCount(count + 1); // Potential stale state issue
```

✅ Use functional updates:

```jsx
setCount((prevCount) => prevCount + 1);
```

### 2. Using `useEffect` incorrectly

#### Example: Missing dependencies leading to stale closures

```jsx
useEffect(() => {
  fetchData();
}, []); // What if fetchData depends on props?
```

✅ Ensure dependencies are correct, or use a callback inside `useEffect`.

```jsx
useEffect(() => {
  fetchData(someProp);
}, [someProp]);
```

### 3. Unnecessary re-renders due to object/array dependency in `useEffect`

#### Example: Objects as dependencies causing frequent updates

```jsx
useEffect(() => {
  fetchData();
}, [filters]);
```

✅ Use `useMemo` or `useCallback` to prevent unnecessary re-renders.

```jsx
const memoizedFilters = useMemo(() => filters, [filters]);
useEffect(() => {
  fetchData();
}, [memoizedFilters]);
```

### 4. Mutating state directly

```jsx
const [user, setUser] = useState({ name: 'Alice', age: 25 });
user.age = 26; // ❌ This won't trigger a re-render
```

✅ Use a new object reference:

```jsx
setUser((prevUser) => ({ ...prevUser, age: 26 }));
```

### 5. Not cleaning up side effects in `useEffect`

#### Example: Memory leaks due to missing cleanup

```jsx
useEffect(() => {
  const interval = setInterval(() => {
    console.log('Running...');
  }, 1000);
}, []); // No cleanup
```

✅ Always return a cleanup function when needed:

```jsx
useEffect(() => {
  const interval = setInterval(() => {
    console.log('Running...');
  }, 1000);
  return () => clearInterval(interval);
}, []);
```

## What you need to know for interviews

1. **Explain the purpose of hooks** – Why React introduced hooks and how they replace class component lifecycle methods.
2. **Know the common hooks** – Understand when and why to use `useState`, `useEffect`, `useContext`, and `useRef`.
3. **Understand re-rendering issues** – Recognize how dependency arrays impact performance and how `useMemo`/`useCallback` optimize rendering.
4. **Implement a custom hook** – Be able to write and explain a reusable custom hook in an interview.
5. **Debug common hook pitfalls** – Identify and fix common mistakes like missing dependencies in `useEffect` or unnecessary state updates causing re-renders.

React hooks, with its restrictions can seem unintuitive at first. The `useEffect` hook is especially hard to get right, even for seasoned developers. Our recommendation is to avoid using `useEffect` if there are alternatives.

Given interview questions are mostly self-contained, `useEffect`s aren't too commonly needed. Hence you will find that the official solutions by GreatFrontEnd's React user interface questions do not have too much `useEffect` usage. If you are using `useEffect` in your code, compare with the official solutions to see how you can possibly avoid using it.
