title: Common React mistakes made during Interviews
description: TODO
---

- Passing setState into child components
- Not cancelling async requests upon unmount
- useEffect shenanigans
- Using index as key
- Unnecessary state when they can be derived
- Direct mutation of state/props
- Calling a function when passing into props
- Using non-interactive elements for interaction
- Calling logic within useEffect vs within event handlers
- Not cleaning up at the end of an effect
